Имя устройства
Дирректория текущая - ll
Корневая директория - /
Относительный путь - путь от текущей директории 
Абсолютный путь - это путь, который начинается со /
Преобразование от относительного до абсолютного пути происходит путем добавления к нему текущего адреса

Команды: 

ls- команда, которая показывает список файлов в дирректории

ll = ls -alhF

cd - заменяет текущую дирректорию, на которую указываешь


pwd - команда, которая показывает абсолютный путь до текущей дирректории

mkdir - создает директорию внутри текщей директории с указанным названием

touch - создает файл внутри текщей директории с указанным названием

cat - возвращает содержимое файла в консоль 

man - выводит описание команды(мануал)

vim - текстовый редактор в консоли.
    i - даёт доступ писать в файле.
    esc - выходит из этого режима.
    :wq - выйти и сохранить
    :q - выйти
    dd - вырезать строчку
    yy - копировать строчку
    [N]dd ([N]yy) - вырезать(скопировать) N строк 
    P - вставить перед текущей строкой
    p - вставить после текущей строки

history - выводит историю команд терминала

head - команда которя выводит первые 10 строк файла
    можно через -n поменять на другое число
    Пример:
        head -n 20 file_name.txt

tail - то же самое что и head, но с конца файла

grep - команда, если которую добавить через |(пайп), выведет только те строчки, в которых содержится введенное.
    Можно записать и без пайпа grep 'какое-то выражение'(файл)

top - показывает активные процессы в системе

kill - посылает процессу с pid(process identificator) сигнал 
    sigterm - если не указан номер сигнала при kill
    sigint - посылается когда в терминале нажимаешь ctrl+c. Останавливает программу
    sigkill - посылается при kill -9 (9 - номер сигнала sigkill)
    Посмотреть номера сигналов можно через `man signal`

ps aux - команда, которая показывает активные процессы

which - показывает абсолютный путь до места хранения команды(бинарного исполняемого файла)
    Примеры:
        - which python3
        - which cat

сlear - очищает экран терминала

date - показывает текущее время на устройстве

time - показывает время выполнения команды

wc - команда, которая выводит количество строк, слов и байтов, которые содержатся в файле
    -c - только байты
    -l - только строки
    -m - сколько символов

uname - выводит в терминал имя операционной системы

whoami - выводит имя пользователя

&& - ставится между командами, если левая команда завершилась с ошибкой - не будет запущена правая

; - ставится между командами, если левая команда завершилась с ошибкой - все равно попытается выполнить команду справа

rm - команда, удаляющая файл, если указать путь до файла. Если добавить -r после rm, то удалится все, что внутри дирректории

echo - возвращает в консоль введенный текст. Если поставить >, то выходные данные будут записываются в файл, указанный после >

| - все что выводит первая команда, используется как входной файл для второй команды (не нужно вводить имя файла еще раз)

rmdir (дирректория) - удаляет дирректорию. Не удаляет, если не пустая.

cp - копирует первый файл из одного адреса в другой адрес
    -R делает рекурсивно (Полное копирование папок, смотрит все поддиректории внутри)

ln - s - символическая ссылка(ярлык) на файл 1 создается симлинк на файл 2

mv - команда, которая переносит файл по другому адрасу, если перенести с новым названием - переименуется

git - система контроля версий.
    Нужна для:
        - для сохранения кода(durability)
        - возможность версионирования(откатывать, менять версии) всего проекта
        - возможность совместной работы над проектом
    Основные команды:
        - git status - посмотреть текущий статус репозитория(какие файлы отслеживаются, какие нет, текущая ветка)
        - git add - добавляет файлы в индекс(помечает, какие измененяемые файлы будут добавлены в коммит)
        - git commit - фиксирует текущее состояние изменений индексируемых файлов 
            -m - опция, добавляющая описание к изменению
        - git remote add origin <URL> - делает алиас на origin по урлу
        - git push origin <branch_name> - загружает текущие изменения в удаленный репозиторий в указанную ветку
        - git log - показывает порядок коммитов и кто их делал
        - git diff - показывает изменения
        - git pull origin <branch_name> - забрать изменения
        - git clone - склонировать чужой репозиторий
